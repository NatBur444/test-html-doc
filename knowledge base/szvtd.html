Отчетность в ПФР по форме СЗВ-ТД¶

  • Что нужно сделать, чтобы начать отправлять сведения в ПФР?

  • Порядок действий для отправки заявления на подключение к ЭДОК

  • Порядок действий для отправки отчета по форме СЗВ-ТД

Что нужно сделать, чтобы начать отправлять сведения в ПФР?¶

Справочная информация об отчетности по форме СЗВ-ТД.

Чтобы работодатель мог отправлять в электронном виде сведения в ПФР, ему нужно направить в электронном виде заявление на подключение к системе электронного документооборота ПФР. Данное заявление должно содержать:

  • регистрационный номер страхователя в ПФР;

  • наименование страхователя (юридического лица);

  • ИНН;

  • КПП страхователя (юридического лица);

  • ФИО (при наличии) страхователя (физического лица);

  • реквизиты оператора, через которого планируется передача сведений.

После отправки заявления на подключение к ЭДО ПФР, у работодателя появится возможность сдавать сведения по форме СЗВ-ТД через оператора связи привычным для него способом. Но сначала нужно дождаться успешного принятия заявления на подключение к ЭДО от ПФР. Для отключения от ЭДО ПФР необходимо подать «Заявление на отключение от ЭДО ПФР», пока нет возможности отправить его через API Контур.Экстерна.

Особенности подписи

Заявление и отчет должны быть подписаны подписью в формате XMLDsig. Подробнее в Подпись XMLDsig для отчетов в ПФР.

Порядок действий для отправки заявления на подключение к ЭДОК¶

Заявление на подключение страхователя к ЭДО ПФР можно передать через API Конутр.Экстерна, тип документооборота urn:docflow:pfr-ancillary. Статусы и порядок документооборота описаны в спецификации.

Если есть файл заявления на подключение к ЭДОК с прикрепленной подписью xmlDsig¶

  1. Создать черновик. POST Draft.

  2. Загрузить в сервис контентов подписанный файл заявления с прикрепленной подписью xmlDsig. POST Upload.

  3. Создать документ в черновике, в теле запроса передать идентификатор контента content-id. POST Document.

  4. Проверить, подготовить, отправить черновик. POST Check, POST Prepare, POST Send.

  5. ПФР в течение двух рабочих дней направит «Уведомление о результате рассмотрения», успешно принятым считается уведомление с отметкой об удовлетворении заявления. Документооборот завершен.

Если нет файла заявления на подключение¶

Если нет файла заявления на подключение, его можно сгенерировать при помощи методов формирования файлов в черновике. Сценарий работы с черновиком в таком случае будет следующий:

  1. Создать черновик. POST Draft.

  2. Создать документ в черновике по json контракту. POST BuildDocumentContent.

  3. Добавить в документ подпись XMLDsig:

    1. По content-id скачать сгенерированный файл заявления из сервиса контентов. GET Download.

    2. Добавить в файл подпись в формате xmlDsig. Если нужна помощь в создании подписи, смотри алгоритм формирования XMLDsig подписи на стороне API. Если воспользовались алгоритмом, вернуться в п.6.

    3. Подписанный файл нужно загрузить в сервис контентов. POST Upload.

    4. Заменить контент документа на полученный идентификатор сервиса контентов PUT Document.

  4. Проверить черновик POST Check.

  5. Подготовить черновик POST Prepare.

  6. Отправить черновик POST Send.

Порядок действий для отправки отчета по форме СЗВ-ТД¶

Отчет по форме СЗВ-ТД нужно отправлять отдельным документооборотом типа urn:docflow:pfr-report. Статусы и порядок документооборота описаны в спецификации.

  1. Создать черновик. POST Draft.

  2. Загрузить в сервис контентов подписанный файл отчета с прикрепленной подписью xmlDsig. POST Upload.

  3. Создать документ в черновике, в теле запроса передать идентификатор контента content-id. POST Document.

  4. Проверить, подготовить, отправить черновик. POST Check, POST Prepare, POST Send.

  5. ПФР в течение одного рабочего дня направит «Уведомление о доставке».

  6. ПФР в течение трех рабочих дней с момента отправки «Уведомления о доставке»:

    1. в случае выявления ошибок сформирует и пришлет «Уведомление об отказе в приеме пакета». Документооборот завершен.

    2. по результатам проверок сформирует «Протокол проверок» или «Уведомление об устранении ошибок и (или) несоответствий между представленными страхователем сведениями и сведениями, имеющимися у ПФР».

  7. Оператор ЭДО сформирует «Уведомление о доставке» на «Уведомление об устранении ошибок…», которое будет отправлено в ПФР, а также появится в документообороте. Документооборот завершен.
